# Private memos (not published)

This folder is a personal holding pen for source material — research-paper memos, raw notes, draft fragments, anything in-progress that may later become a blog post, research entry, talk, or report on the public site.

## Why the underscore?

`_memos/` starts with an underscore, which is Quarto's convention for "ignore this folder." `quarto render` skips it entirely, so nothing here is ever generated into `docs/` or published to `adamsceballos.com`. Same convention as `_templates/`, `_freeze/`, and `_extensions/`.

The folder *is* tracked in git, so it syncs across your machines via the GitHub repo — but it stays out of the deployed site.

## What to put here

- **Research-paper memos** generated by the `/research-paper-memo` skill (faithful, source-pinned extractions of papers, working papers, reports, PDFs).
- **Blog drafts** before they're ready to publish — keep them here while iterating, then move (and rename) into `blog/YYYY/MM/<slug>/index.qmd` when you're ready to ship.
- **Loose notes** that may seed future research entries, talks, or CV updates.
- **Outlines, references, quotations** you want to keep close to the project but not on the public site.

## What NOT to put here

- Anything you want on the public site — that goes in `blog/`, `research/`, `reports/`, `talks/`, or `cv/`.
- Secrets, credentials, embargoed work that shouldn't even be in a public GitHub repo — this folder is private to the *deployed site*, not to the world. The GitHub repo is public, so anyone can read these files there.

## Naming convention

Match the `/research-paper-memo` skill's pattern when applicable:

```
author_year_short_title_memo.md
```

For blog drafts and looser notes, anything readable is fine — e.g. `2026-05-draft-trade-costs-blog.md` or `notes-fuzzy-logic-for-blog.md`.
